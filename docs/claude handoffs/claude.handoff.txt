HARD RULES
-------------------------------------------------------------
- .gd files: ASCII ONLY. Non-ASCII causes SILENT parse failures.
- TABS, never spaces. CRLF line endings.
- COMPLETE drop-in files, never diffs. Josh does not hand-edit.
- INSTALL BY REPLACING THE FILE IN FILE EXPLORER, not by pasting into
  the Godot script editor -- see INCIDENTS #5.
- Multi-file deliveries are a MATCHED SET. A "function not found in
  base CraftingData/CraftingService" error means a file did not get
  copied, not a code bug.
- Before every export: ASCII check, tab check, undeclared-identifier
  sweep, and for large refactors a DECLARATION COUNT comparison.

CRAFTING SYSTEM
-------------------------------------------------------------
DESIGN DECISIONS
  - Plain Dictionaries, NOT Resource/.tres. Save file is JSON, so they
    round-trip with no conversion layer.
  - Material Quality is 0-100. The legacy weapon "Quality" is 0-1000.
    QUALITY SCALE BRIDGE in _grant_crafted_item multiplies by 10. Do
    not "simplify" that away or every crafted weapon sits at the bottom
    of its stat ranges.
  - 4-of-15 keystone selection is CRAFTING ONLY. Combat keystones keep
    the point-budget model.
  - The crafting keystone gets its OWN panel, separate from the talent
    viewer.
  - Every crafting keystone node needs a DOWNSIDE (Material Insight and
    Precision Calibration as specced are pure upside = auto-picks).

PHASE 1 - DATA FOUNDATION (done)
  Material batches, crafted item instances, player profile, all with
  factories and to_dict/from_dict. craft_seed stored from day one.
  Provenance (source_id / floor_id / location_id) on every batch.

PHASE 2 - CAMPAIGN RESOURCE GENERATOR (done)
  60 floors: Survivors 1-20 (quality bias -8), Expansion 21-40 (0),
  Builders 41-60 (+12). Deeper is OLDER but cleaner. 15 floor
  identities, 22 resource families. Same seed -> identical map;
  reload never rerolls; guarantees always satisfiable via deterministic
  repair. NOTE: guarantees are met naturally on only ~33% of seeds;
  the rest get repaired. Placeholder numbers, needs playtesting.

PHASE 3 - BASIC CRAFTING (done, wired in)
  Alley dumpster yields real MaterialBatches. Blueprint panel with
  material slots. Medicine blueprints (bp_bandages, bp_antiseptic_salve)
  restored medicine after the rip-out.

PHASE 4 - EXPERIMENTATION (done, wired in)
  8 categories (Output, Precision, Efficiency, Thermal Control,
  Durability, Handling, Potency, Mod Architecture), each mapped to a
  stat and to the material traits that help it.

  Point generation: Base 3 + familiarity 0-3 + materials 0-2
  + workshop 0-2 (Phase 8) + tools 0-2 (Phase 8) + keystone 0-3
  (Phase 7). Workshop/tool/keystone all contribute 0 today, so the
  ceiling rises automatically when those land.

  Result tiers: Breakthrough 95+ (x1.6), Exceptional 82 (x1.3),
  Great 68 (x1.0), Good 52 (x0.75), Standard 35 (x0.5), Poor 18
  (x0.25), Failure below (x0.0).

  Risk modes: Stable (+5 score, +/-8 spread, 0% flaw chance),
  Standard (0, +/-15, 25%), Aggressive (+12, +/-30, 55%).

  FAILURE COSTS THREE THINGS (Josh's call): points wasted, a
  risk-scaled chance of a permanent instability on the item, and a
  socket_chance_modifier penalty read by Phase 5. Failed categories
  also cost 8 durability each.

  ANTI-SAVE-SCUM: craft_seed is DERIVED from blueprint + sorted material
  batch ids, NOT randomised. Reload + same materials + same allocation
  + same risk = identical result. Allocating DIFFERENTLY gives a
  different result, which is correct -- that is a choice, not a reroll.

  TUNING NOTE: concentrating points in one category yields more total
  benefit than spreading them. Real choice, but spreading is currently
  the weaker play rather than a genuine alternative.

PHASE 5 - MOD SOCKETS (done, wired in)
  Opportunity score feeds four probability bands:
    low <25       90/10
    moderate 25-49  55/40/5
    strong 50-74    20/60/18/2
    exceptional 75+  5/45/40/10
  Mod Architecture allocation is the dominant term (x12 per point),
  plus tier multiplier x10, familiarity, material compatibility,
  workshop; minus difficulty x0.20 and 5 per instability, plus the
  Phase 4 failure penalty. Rolls derive from craft_seed via an
  independent derivation, so reopening the UI cannot reroll.
  Consumables declare max_sockets 0 and are skipped.

PHASE 6 - MODS (LOGIC BUILT, NOT WIRED IN)
  BATCH 1 SHIPPED: 8 weapon mods, 5 grades, install rules, stat
  effects. Scope is WEAPON mods only -- the spec also lists Armor and

  Grades: Standard 1.0, Refined 1.4, Advanced 1.8, Prototype 2.3
  (drawback x1.6), Masterwork 2.1 (drawback x0.0). Masterwork gives
  LESS raw power than Prototype but removes the penalty, so the top

-------------------------------------------------------------
KEY INTEGRATION POINTS IN main.gd
-------------------------------------------------------------
_realise_item_stats(item_key, definition, quality)
  SHARED by _grant_starting_weapon and _grant_crafted_item. Reads
  weapon_stat_ranges and scales each stat by quality, computes DPS.
  Extracted because crafted weapons were previously granted with ONLY
  a Quality value -- no damage, speed, accuracy or damage type. They
  were blank items dealing the fallback 5 damage. Keep both grant
  paths going through this so they cannot drift again.

crafted_item_instance_of  inventory item_key -> crafted instance_id.
  Lets the inventory detail panel show traits, flaws, socket types and
  durability. Persisted in save/load.

_scavenge_dumpster()  draws from EVERY live surface source, not just
  the ones located at "Alley Dumpster". Copper (Stripped Wiring) and
  black iron (Rusted Railing) exist in SURFACE_SOURCES but have no
  physical props, so they were unreachable. When props are added,
  split them back into separate interactables.

KNOWN DEBTS AND LIMITATIONS
-------------------------------------------------------------
TESTING VALUES LEFT IN (restore before any balance pass)
  - DUMPSTER_RESPAWN_TIME = 5.0, was 45.0. At 5s the surface salvage
    economy is meaningless. Josh asked to leave it for now.

CONSUMABLES DO NOT SCALE WITH QUALITY
  Bandages heal a flat BANDAGE_HEAL_AMOUNT (100) regardless of craft
  quality, and the heal constant is used directly at the point of use.
  So Efficiency and Potency experimentation do NOTHING on the item
  players craft most. Blood Bag already scales via _scale_by_quality --
  the machinery exists, bandages just do not use it. Fix: give bandages
  a min/max heal range and have Efficiency reduce BANDAGE_ACTION_COST.

BATCH POOLING
  A material slot must be filled from ONE batch (_batches_for_slot
  requires a single batch with amount >= needed). Batches only merge
  when family, quality, trait, instability AND source all match. In the
  Silo, where many sources of the same family exist, a player will be
  able to carry plenty of a material and still not fill a slot. The
  panel now says "not enough in one batch -- N held across all
  batches" so it is at least honest. Options: leave it (one lot per
  slot is defensible), allow pooling with averaged quality (preferred,
  needs multi-select UI), or merge more aggressively (destroys
  provenance, which the spec wants).

LOOT STILL DROPS INERT MATERIALS
  Loot tables drop Copper/Iron etc. from the old system. Those items
  have no consumer now. Data debt, cheap to fix once the new material
  model is settled.

ORPHANED ITEMS IN OLD SAVES
  Mineral Survey Tool, Rusty Crafting Kit etc. remain in inventories
  from before the rip-out. Their ITEM_DEFINITIONS were removed so they
  do nothing.

INCIDENTS AND LESSONS
-------------------------------------------------------------
1. GAMEDATA DESYNC. A stale GameData.gd on both machines carried
   pre-4b damage type names. Twelve weapons matched no resistance key,
   got rating 0, and bypassed enemy armor entirely for several sessions
   while the system was being built on top of. Failed silently because
   a missing key returns a legal zero.
   LESSON: install on both machines the same day. The startup guard now
   catches this class of bug, but only AFTER the fact.

2. FUNCTION-SPAN DELETION (happened TWICE). A bulk edit defined a
   function's end as the NEXT `func` keyword, swallowing any top-level
   declaration block between them. Once ~725 lines and 30+ functions;
   once 80 declarations including BASE_HIT_CHANCE, MELEE_RANGE,
   inventory, equipped_weapon_name. Both recovered from a pre-edit
   backup.
   LESSON: a function ends at the first COLUMN-0 non-blank line. Back
   up before bulk edits. Compare DECLARATION COUNT before and after --
   that is the check that catches it immediately. Dry-run large cuts.

3. LOST DECLARATION AFTER RECOVERY. Restoring from a snapshot rolled
   back a `var enemy_nodes` declaration while the redo rebuilt only the
   functions USING it -- 9 "not declared in current scope" errors.
   LESSON: after recovering, re-check that every declaration the
   refactor introduced still exists.

4. RIGHT_X ORPHAN. A refactor REMOVED a declaration while two child
   controls still referenced it.
   LESSON: grep for users before deleting a declaration.

5. PASTE-INSTEAD-OF-REPLACE. main.gd ended up containing the new file
   followed immediately by the old one ("...visible = falseextends
   Node2D"), producing 447 errors. The delivered file was clean; the
   corruption happened at install.
   LESSON: install by REPLACING THE FILE in File Explorer. Do not paste
   contents into the Godot script editor -- the paste lands at the
   caret rather than overwriting.

6. UI BUILT WITH ABSOLUTE OFFSETS. The crafting panel was positioned
   with pixel offsets measured up from the bottom of the screen, then
   with full-screen containers. The first broke at any viewport height
   other than the one assumed; the second smeared sparse content across
   1080px. Third attempt -- a FIXED-SIZE CENTRED window (1180x764) --
   works.
   LESSON: for code-built panels, use a fixed-size centred frame with
   containers inside. There is a comment in CraftingPanel._build_ui
   recording this so it does not get "fixed" back.

7. LABEL TOOLTIPS SILENTLY DO NOTHING. Godot Labels default to
   MOUSE_FILTER_IGNORE, so they never receive hover and tooltip_text is
   never shown. Category tooltips existed for a whole phase without
   ever appearing.
   LESSON: set mouse_filter = MOUSE_FILTER_STOP on any Label carrying a
   tooltip.
