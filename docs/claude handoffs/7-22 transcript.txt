   walking height. Added StaticBody3D + BoxShape3D collision to
   all 5 street kit pieces:

   - SteamtekSidewalkCurbRamp2_4m3D.tscn: Multiple shapes including ConvexPolygonShape3D ramp

   First attempt crashed Godot -- sub_resource blocks were placed
   after node definitions in the .tscn files. Godot requires ALL
   sub_resources before ANY nodes. Fixed by reordering all 5 files.

3. RAMP COLLISION ATTEMPTS (multiple failures):
   User reported "the ramp collision is off. I cant walk up it from
   any side" and later "I dont want to use the curbed opening. I want
   to be able to walk up onto the sidewalk wherever i want."

   Attempted solutions that FAILED:
   a) ConvexPolygonShape3D ramps -- too steep (42-50 degrees), exceeded
      CharacterBody3D's 45-degree floor_max_angle. Jolt Physics rejected them.
   b) Rotated BoxShape3D ramps -- worked in one curb direction but not
      the other (curbs get rotated 90 degrees differently in the scene).
   c) Tent/dual ramps -- added ramps from both sides. Still didn't solve
      sidewalk approach.
   d) Even 0.04m thin slabs created vertical edges that move_and_slide()
      treated as walls.

4. STEP CLIMBING SOLUTION:
   Final solution: character controller step-up teleport system.
   Added to steamtek_humanoid_character_3d.gd:
   - @export var step_height := 0.25
   - _try_step_up() method: tests if space above is clear, tests if
     floor exists ahead, then teleports character up by step_height
   - floor_snap_length = 0.3 added to CharacterBody3D .tscn to snap
     character down after teleport

   First attempt failed -- gravity was too weak (-0.288 m/s) to reach
   surface in one frame after teleport. Character floated, failed
   is_on_floor(), got reset. Fixed with floor_snap_length + simplified
   teleport without extra move_and_slide call.

   User confirmed: "its working now"

5. F8 QUIT BEHAVIOR:
   User reported "when i F8 its making godot quit" then later "F8

   - get_tree().quit() was closing the game process. When launched from

   - Removed all F8 handlers -- then user couldn't close standalone game.
   - Restored F8 quit in base script (steamtek_transition_level_3d.gd)
   - Added super._unhandled_input(event) calls in apartment and lantern
     scene overrides so F8 reaches the base class handler.
   - steamtek_main_menu.gd has its own F8 handler (doesn't extend base).
   - scenes/main.gd F8 handler removed (replaced with pass).
